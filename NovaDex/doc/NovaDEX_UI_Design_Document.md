# NovaDEX UI 设计文档

## 1. 设计概述

NovaDEX 是一个现代化的去中心化交易所，专注于提供流畅、安全、高效的交易体验。UI 设计遵循以下核心原则：

- **现代简洁**：采用清晰的视觉层次和简洁的设计语言
- **用户友好**：直观的界面布局，降低用户学习成本
- **专业可信**：通过设计传达安全性和可靠性
- **响应式设计**：适配各种屏幕尺寸和设备
- **高性能**：优化动画和交互，确保流畅的用户体验

### 1.1 设计风格

- **主题**：深色主题为主，减少视觉疲劳，突出关键信息
- **风格**：现代、科技感、专业
- **关键元素**：卡片式布局、微妙的渐变效果、平滑的过渡动画

## 3. 排版系统

### 3.1 字体

- **主要字体**：Inter（无衬线字体）
- **备用字体**：Roboto, Arial, sans-serif

### 3.2 字体层次结构

| 级别 | 字体大小 | 字重 | 行高 | 用途 |
|------|----------|------|------|------|
| H1 | 32px | 700 | 40px | 页面标题 |
| H2 | 24px | 600 | 32px | section标题 |
| H3 | 20px | 600 | 28px | 卡片标题 |
| H4 | 18px | 600 | 24px | 组件标题 |
| 正文 | 16px | 400 | 24px | 主要文本内容 |
| 辅助文本 | 14px | 400 | 20px | 辅助信息、说明文本 |
| 小字 | 12px | 400 | 16px | 标签、状态文本 |
| 按钮文本 | 16px | 600 | 24px | 按钮文字 |

## 4. 间距系统

采用 8px 网格系统，确保界面元素间距的一致性：

| 间距值 | 用途 |
|--------|------|
| 4px | 细微间距、图标与文字间距 |
| 8px | 小间距、表单元素间距 |
| 12px | 中小组件内部间距 |
| 16px | 标准间距、卡片内部间距 |
| 24px | 大间距、区域分隔 |
| 32px | 页面主要内容间距 |
| 48px | 大型区块间距 |

## 5. 图标系统

- **风格**：线性图标，简洁现代
- **尺寸**：16px, 20px, 24px, 32px
- **颜色**：与文本颜色保持一致，或使用主题色作为强调

## 6. 组件设计原则

### 6.1 按钮

#### 主要按钮
- 背景：主渐变
- 文字：白色
- 圆角：8px
- 内边距：12px 24px
- 悬停效果：轻微放大 (1.02x)，阴影增强
- 点击效果：轻微缩小 (0.98x)

#### 次要按钮
- 背景：透明
- 边框：主色
- 文字：主色
- 圆角：8px
- 内边距：12px 24px
- 悬停效果：背景色变为主色的10%透明度

#### 禁用按钮
- 背景：中性色（#334155）
- 文字：文本弱
- 圆角：8px
- 内边距：12px 24px
- 不可交互

### 6.2 输入框

- 背景：卡片背景色 (#1E293B)
- 边框：边框色 (#334155)
- 文字：文本主色
- 圆角：8px
- 内边距：12px
- 聚焦效果：边框变为主色，添加轻微阴影
- 悬停效果：边框颜色变亮

### 6.3 卡片

- 背景：卡片背景色 (#1E293B)
- 边框：边框色 (#334155)
- 圆角：12px
- 阴影：轻微阴影 (0 4px 6px -1px rgba(0, 0, 0, 0.3))
- 内边距：16px
- 悬停效果：阴影增强，轻微上浮

### 6.4 导航

- 背景：背景色 (#0F172A)
- 文字：文本主色
- 激活状态：文字变为主色，底部添加主色边框
- 悬停效果：文字颜色变亮

### 6.5 加载状态

- 使用骨架屏表示内容加载中
- 按钮和操作元素使用加载指示器
- 加载动画应平滑、不突兀

## 7. 页面布局原则

### 7.1 整体布局

- **桌面端**：三栏布局（侧边栏、主内容区、辅助信息区）
- **平板端**：两栏布局（可折叠侧边栏、内容区）
- **移动端**：单栏布局（顶部导航、内容区）

### 7.2 响应式断点

| 设备 | 断点 |
|------|------|
| 移动端 | < 768px |
| 平板 | 768px - 1024px |
| 桌面 | > 1024px |

### 7.3 页面结构

每个页面应包含以下基本结构：

1. 顶部导航栏
2. 侧边栏（桌面端）
3. 主内容区
4. 页脚（可选）

## 8. 页面设计

### 8.1 交易页面 (Swap)

- **主要功能**：代币交换
- **布局**：
  - 左侧：交易表单（输入/输出代币、数量）
  - 右侧：价格图表、订单簿、交易历史
  - 底部：交易按钮、滑点设置

### 8.2 流动性页面 (Liquidity)

- **主要功能**：提供和管理流动性
- **布局**：
  - 顶部：添加流动性表单
  - 中间：流动性池列表
  - 底部：管理现有流动性

### 8.3 头寸页面 (Positions)

- **主要功能**：管理流动性头寸
- **布局**：
  - 顶部：头寸列表
  - 中间：选中头寸的详细信息
  - 底部：管理头寸操作（增加、减少、关闭）

### 8.4 池探索页面 (Pool Explorer)

- **主要功能**：浏览和搜索流动性池
- **布局**：
  - 顶部：搜索和筛选功能
  - 中间：池列表（包含池信息、TVL、交易量）
  - 底部：分页控件

## 9. 交互设计

### 9.1 动画与过渡

- 所有状态变化应使用平滑过渡（200-300ms）
- 悬停效果应即时响应
- 页面切换使用淡入淡出效果
- 模态框从中心弹出

### 9.2 用户反馈

- 操作成功/失败应显示明确的通知
- 长时间操作应显示加载指示器
- 表单验证错误应实时显示

### 9.3 可访问性

- 所有交互元素应具有足够的点击区域
- 支持键盘导航
- 提供清晰的视觉反馈
- 确保文本与背景的对比度符合 WCAG 标准

## 10. 前端技术实现

### 10.1 技术栈

- **框架**：React 18 + TypeScript
- **样式**：Tailwind CSS 3
- **状态管理**：Redux Toolkit
- **Web3 集成**：Ethers.js 6
- **图表库**：Chart.js
- **构建工具**：Vite

### 10.2 性能优化

- 组件懒加载
- 图片优化
- 减少不必要的重渲染
- 优化动画性能
